# 01 · Problem & Context

## 🌍 Business Context

一个北美 AI 教育出海产品的运营模式：
- **200+ 北美校园大使**每周拍视频
- **10 人内部 Coach 团队**负责对接 + 内容指导
- 每周大使发视频 → 跑数据 → **按视频数 × 单价 + 播放量奖励 + 调整项**计算薪酬
- **每周通过美区薪酬平台跨境打款**

每周覆盖 **~200 笔结算**。

## 🎯 The Concrete Problem

### Painpoint A · Coach 周结算耗时 ~10 小时

原本流程：

```
Coach 每周：
  ① 拉数据：从内部系统 / Excel 把每位下属的视频数、播放量拉下来
  ② 手算：视频数 × 单价 + 播放量奖励 + 各种调整项
       → 一位大使 5-10 分钟
       → 30 位下属 × 5 分钟 = 2.5 小时（这还是顺利的情况）
  ③ 核对：手算的数字和系统数据对一下，找差异
  ④ 内部审批：跟内部财务 / 主管 review
  ⑤ 到美区薪酬平台：一条一条输入金额 / 选择 Creator / 点确认
       → 30 位 × 1-2 分钟 = 30-60 分钟
  ⑥ 二次核对：打款记录 vs 系统记录对账

每周一位 Coach: 大约 8-10 小时
10 位 Coach × 8 小时 = 团队每周 ~80 小时被结算吃掉
```

**这是高度可自动化的工作，但人在做。**

### Painpoint B · 错误率高

人工 Excel 计算 + 人工逐条输入 → **错误率 10-20%**：
- 漏算某条视频
- 单价用错档位
- 奖励规则记错
- 调整项忘加

每次出错 = 大使来投诉 = Coach 还要重新跑一遍流程。

### Painpoint C · 跨境合规复杂

美区薪酬平台 ≠ 国内的薪资 SaaS。涉及：
- **税务文件**（W-9 / W-8BEN）需要 Creator 自己填
- **银行账户验证**周期长
- **跨境支付到账时间**不固定
- **US Creator vs Non-US Creator** 走不同合规流程

→ 这部分**不能完全去人工化**（合规风险），但可以**半自动化**。

### Painpoint D · 内部审批拖慢节奏

原本每周结算前要走内部审批流：
- Coach 算完 → 主管 review → 财务 approve → 打款
- 每一层都可能找几天才有空

但实际上 **99% 的结算是规则化的**（没什么好审的）。审批流变成了纯粹的延迟来源。

## 🔍 What I Set Out To Build

不是"做一个 Excel 计算器"，而是一套**端到端的结算自动化产品**：

1. **自动算薪**：取代人工 Excel 计算
2. **Creator 自助确认**：让大使自己 review 数字，减少争议
3. **超期废弃机制**：避免薪酬挂账长期不处理
4. **半自动打款**：CSV 一键导入美区薪酬平台
5. **取消大部分内部审批**：直发为主，仅特殊情况加 review

## 🧠 Key Design Decisions

| 决策 | 理由 |
|------|------|
| **结算周期窗口** vs 按月 | 周期窗口更灵活，且能匹配实际业务节奏 |
| **Creator 自助确认** vs Coach 单方说了算 | 减少 80% 的薪酬争议 |
| **超期自动废弃** vs 无限期等确认 | 防止挂账无限期累积，倒逼大使处理 |
| **CSV 半自动打款** vs 全 API 自动 | 合规要求保留财务"最后一道把关"——CSV 是最优解 |
| **取消大部分内部审批** vs 维持多层 review | 99% 结算是规则化的，审批是纯延迟 |
| **Geo 分流** US 自动 / Non-US 手动 | 跨境合规复杂，Non-US 国家的支付走兜底流程 |
