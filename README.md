# Internal Payroll Automation

> 自动化的跨境薪酬结算系统 · Coach 周结算耗时从 _~10 小时 → 几分钟_ · 对接美区薪酬平台半自动批量打款

🎯 这是 [Creator Ops Portfolio](https://github.com/Vikki-L/creator-ops-portfolio) 的精选独立项目。

---

## TL;DR

**Problem**：一个北美 AI 教育出海产品的 Coach 每周需要在 Excel 里手算每位大使的薪酬（视频数 × 单价 + 播放量奖励 + 调整项）+ 人工逐条到美区薪酬平台上输入金额、点确认。**一位 Coach 每周仅结算就要约 10 小时**，且容易出错。

**Approach**：设计自动化跨境薪酬结算后台，含**结算周期窗口** + **自动算薪** + **Creator 自助确认** + **超期废弃机制** + **美区薪酬平台半自动批量打款**（CSV 一键导入）。

**Result**：

- ✅ 每周覆盖 **~200 笔结算**
- ✅ Coach 单周结算耗时 _~10 小时 → 几分钟_
- ✅ 人工错误率下降约 **10-20%**
- ✅ 打款步骤从「逐条手动输入」→「CSV 一键导入」

## 📐 System Highlights

### 核心设计模块

| 模块 | 解决什么 |
|------|---------|
| **结算周期窗口** | 把"按月结算" → "按周期窗口结算"，每位 Coach 看到的窗口清晰 |
| **自动算薪** | 视频数 × 单价 + 播放量奖励 + 调整项 → 系统自动算（替代人工 Excel）|
| **Creator 自助确认** | 让大使在自己账号确认薪酬明细，**减少结算争议** |
| **超期废弃机制** | 长期不确认自动废弃，避免薪酬挂账无限期累积 |
| **跨境分流**（Geo） | US 走美区主流程 / Non-US 暂手动改状态 |
| **取消内部审批** | 系统直发，仅可加调整项（必填理由），减少人为环节 |
| **半自动打款** | CSV 一键导入美区薪酬平台，替代逐条手动输入 |

### 关键风控设计

- **任意金额奖励/调整必填理由（≥X 字）**：防止滥用
- **超期预警**：在自动废弃前提前预警 Coach
- **分级状态只读**：结算模块只能读分级，不能改（避免越权）
- **跨境状态分流**：US 自动跑，Non-US 走手动兜底（避免合规风险）

## 💡 Key Insights from This Project

1. **跨境薪酬不是单点优化问题**——它涉及计算 / 确认 / 打款 / 容错 / 风控 5 个模块的联动设计
2. **Creator 自助确认是关键风控**——减少 Coach 的"我说了算"造成的争议
3. **超期废弃机制**比"无限期等确认"更健康——倒逼 Creator 及时处理
4. **半自动 vs 全自动**：CSV 一键导入 > 全自动 API（合规 + 财务复核需求）
5. **取消内部审批**这种"减法"往往比"加流程"更有价值
